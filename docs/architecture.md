Тут описание архитектуры проекта, обоснование принятых решений (API vs local inference), план развития до сервиса (API, idempotency, storage, async, мониторинг и т.д.).

Решение построено как двухэтапный пайплайн: ocr_service извлекает текст и confidence из изображения, затем extraction_ervice нормализует текст и по правилам/контекстным маркерам выделяет сущности Фамилия, Имя, Отчество. main.py оркестрирует шаги и возвращает структурированный JSON-результат с диагностической информацией (confidence, ошибки распознавания).

В качестве базового решения выбраны локальные модели, чтобы обеспечить полностью автономный и воспроизводимый запуск без API-ключей и внешних зависимостей. Архитектура построена через абстракции провайдеров OCR и extraction, поэтому при необходимости можно переключиться на API-провайдеров для повышения качества/скорости без изменения бизнес-логики.
